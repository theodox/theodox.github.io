GUI controls

The GUI class defines several types of controls. GUILayout has the same ones, although they take a slightly different set of arguments.

All GUI controls are functions, not objects. That means you have to get the result of any thing like a slider or a text field as you create it, not ‘check it’s properties’ later. This is what triggers all of those redraws.

On Screen items

For a complete list of the GUI building blocks see the GUI and GUILayout docs. These are the most common ones, however:

  • Box Show a text or image
  • Button Creates a button you can click on.
  • HorizontalScrollbar and VerticalScrollbar Shows a horizontal scrollbar
  • HorizontalSlider and VerticalSlider Shows a slider control.
  • Label Draw a read-only string
  • TextArea Draw a multi-line editable text area for editing long text entry.
  • TextField Draw a one line editable text entry field
  • Toggle Creates an on/off button that works like a checkbox (but can have any appearance)

Windows

  • Window Makes a movable window that can contain other controls
  • ModalWindow Shows a dialog-like window that takes over the screen (Users can’t click on the main window)

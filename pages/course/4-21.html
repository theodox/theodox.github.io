GUI: Collecting inputs

If you want to collect an input from a GUI control, you need to have a variable to store the result in. Usually this you want to do this with a class field .

  1. Go back the the RayTest script and add a public field at the top:
    public float Offset = .5f;
  2. Down in the update method, change
    Vector3 targetpos = new Vector3 (hit.point.x, .5f, hit.point.z);
    to 
    Vector3 targetpos = new Vector3 (hit.point.x, Offset, hit.point.z);
    This should let the Offset field control the height of the cube above the mouse location.
  3. Now return to the MouseMoveGUI script in MonoDevelop
  4. Add a field at the top: 
    float Offset = .5f;
    Note that this is not a public field, so it doesn’t show up in the Inspector.
  5. In the OnGUI() method, add two new lines
    Rect slider = new Rect(10, Screen.height - 30, 100, 100);
Offset = GUI.HorizontalSlider(slider, Offset, -1, 2);
  6. Save and run. This should draw a slider at the lower left hand corner. This doesn’t do anything yet, we just want to make sure it works.
  7. You can adjust the position of your slider if you want by changing the first two values in 
    slider = new Rect(10, Screen.height - 30, 100, 100);
  8. Now let’s hook up the control. In the MouseMoveGUI Script, add a new field:
    public RayTest Target;
    This should let you hook up the controlling script on the cube.
  9. In the Update() method, add this: 
    Target.Offset = Offset;
    This should set the offset value in the Raytest script to the value in your slider;
  10. Run the game and play with the slider values to see the effect.

This example shows two common tricks:

  • Using a field to both set the value of a GUI element and also to hold a new value.
  • Using a GUI element to put values into a field in the another object. We could have tried to control the object directly from the GUI script, but that’s a bad idea - the multiple-updates-per-frame could be very costly.
